# 프로젝트: M&A 동향 분석 Agent

## 협업 모드: Claude Code ↔ Codex CLI

이 프로젝트는 **Claude Code**와 **OpenAI Codex CLI** 두 에이전트가 동일한 워킹디렉토리에서 협업합니다. 서로의 작업을 교차 검증하여 보완성을 높이는 것이 목표입니다.

### 역할 분담 (기본 권장)
- **Claude Code (나)**: 설계·리팩토링·문서화·테스트 작성, 한국어 리포트 생성, 큰 컨텍스트 처리
- **Codex CLI**: 알고리즘 구현·최적화·디버깅·대안 구현 제시

> 역할은 고정이 아닙니다. 필요 시 사용자의 지시에 따라 교차 가능.

### 협업 프로토콜

모든 핸드오프는 `.collab/` 디렉토리를 통해 이루어집니다.

#### 1. 핸드오프 보드 (`.collab/handoff.md`)
- 양쪽 에이전트가 작업 상태/요청/응답을 기록하는 공용 게시판
- **항상 새 항목은 파일 맨 위에 추가** (최신순)
- 작성 형식:
  ```
  ## [YYYY-MM-DD HH:MM] FROM: claude TO: codex | TYPE: review-request
  ### 대상: <파일/라인>
  ### 요청: <검증/구현/리뷰 요청 내용>
  ### 컨텍스트: <배경, 제약>
  ### 응답: (상대 에이전트가 채움)
  ---
  ```
- TYPE 종류: `review-request`, `review-response`, `task-handoff`, `question`, `decision-log`

#### 2. 작업 시작 시 필수 절차 (자동 실행)
대화가 시작되면 **사용자 요청 없이도 자동으로** 다음을 수행:
1. `git pull --rebase origin main`으로 최신 코드 동기화
2. `.collab/handoff.md` 상위 항목을 읽어 나(Claude)에게 온 미처리 요청 확인
3. **미처리 요청이 있으면 즉시 처리 착수** — 사용자에게 "할까요?" 묻지 않고 바로 실행
4. 내가 수정할 파일에 상대가 최근 손댄 흔적이 있는지 `git log -n 5 <file>` 확인
5. 충돌 가능성 있으면 사용자에게 먼저 보고
6. 처리 완료 후 결과를 사용자에게 요약 보고

#### 3. 작업 종료 시 필수 절차
의미 있는 변경(코드 수정·신규 파일·리팩토링) 후:
1. `.collab/handoff.md` 상단에 변경 요약 + 검증 요청 사항 기록
2. 사용자에게 "Codex 검증 받기" 옵션 제시
3. 변경된 파일 즉시 commit & push (사용자 별도 요청 불필요 — 메모리 규칙)

#### 4. 검증 요청 패턴
내가 작성한 코드는 가능하면 **Codex에게 리뷰 요청**:
- 잠재적 버그 / 엣지 케이스
- 성능 / 가독성 / 더 나은 패턴
- 누락된 테스트 케이스

상대가 요청한 리뷰는 객관적으로 수행 — 무조건 동의 X, 근거 제시.

### 협업 디렉토리 구조
```
.collab/
├── README.md         # 협업 규칙 (사람이 읽는 용도)
├── handoff.md        # 양방향 핸드오프 보드 (필수)
└── decisions.md      # 두 에이전트가 합의한 결정 로그
```

### 주의사항
- `.collab/` 파일은 양쪽이 동시에 쓸 수 있으므로 작성 직전 `git pull` 권장
- 같은 파일을 양쪽이 동시 수정 금지 — 핸드오프 보드에 "작업 중" 표시
- 사용자가 별도 모드 지정("Claude만", "Codex만") 시 협업 절차 생략 가능
